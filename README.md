# Kardian Daily Checklist

신한펀드파트너스 카디안(Kardian) Daily NAV and Valuation Review 체크리스트를 팀원과 함께 작성하고,
완료된 내용을 PDF로 만들어 고객사에 메일로 보낼 수 있는 설치형 웹앱(PWA)입니다.

원본 `카디안체크리스트.xlsx`의 항목/설명/레이아웃을 그대로 반영했습니다.

## 주요 기능

- **날짜 자동 설정** — 상단 Date는 오늘(정확히는 "현재 작성 중인 영업일")로 자동 표시됩니다.
- **No. of Count** — 숫자만 입력 가능한 입력창.
- **Checking Result (Y/N)** — 기본값 Y, 버튼으로 Y/N 전환. 섹션 제목 행(엑셀의 보라색 행)은 입력란 자체가 없어 값을 넣을 수 없습니다.
- **설명(i) 버튼** — 엑셀 E열에 있던 설명이 있는 항목만 "i" 아이콘이 표시되고, 누르면 설명 팝업이 뜹니다.
- **금일 특이사항** — "금일 특이사항 없습니다." 문구가 기본값으로 들어가 있으며 자유롭게 수정 가능합니다.
- **체크자 / 확인자 서명** — 이름은 키보드로 입력하고, 서명은 은행 서명패드처럼 화면(마우스/터치)에 직접 그릴 수 있습니다.
- **PDF 저장** — 화면에 보이는 항목들을 원본 엑셀과 동일한 레이아웃(Description / No. of count / Checking Result / 설명)으로 재구성해 A4 한 페이지로 캡처합니다.
- **메일 발송** — 버튼을 누르면 PDF가 자동으로 다운로드되고, 받는 사람/제목/본문이 채워진 Gmail 작성 화면이 열립니다. 다운로드된 PDF만 첨부하면 됩니다. (아래 "메일 발송 방식" 참고)
- **자동 마감 / 초기화** — 매 영업일 리포트는 다음 영업일 오전 7시에 자동으로 잠깁니다. 그 전까지는 계속 수정할 수 있고, 잠긴 뒤에는 조회만 가능합니다.
- **과거 3영업일 보관/조회** — 작성 중인 리포트 + 과거 3영업일치, 총 4일분의 리포트를 기기(또는 Firebase 연결 시 팀 전체)에 보관하며, 화면 상단 날짜 버튼으로 넘나들며 조회할 수 있습니다. 그보다 오래된 데이터는 자동 삭제됩니다.
- **로그인 / 사용자 구분** — 앱 접속 시 사번 ID 5자리를 입력해야 들어갈 수 있습니다. 등록된 5명(김광석·남윤호·최한민·박인석·장동현) 중 하나여야 하며, 로그인 정보는 그 기기에 저장되어 다음부터는 자동 로그인됩니다. 상단의 "전환" 버튼으로 다른 사용자로 바꿀 수 있습니다.
- **최종수정자 표시** — 항목의 No. of Count / Checking Result를 수정하면 Checking Result 옆 "최종수정자" 칸에 방금 로그인한 사용자 이름이 자동으로 표시됩니다. 이 칸은 화면에만 보이고, PDF에는 포함되지 않습니다.
- **팀 공유** — Firebase Firestore를 연결하면 오늘 작성 중인 체크리스트를 팀원들이 실시간으로 함께 작성할 수 있습니다(선택 사항). 연결하지 않으면 각자의 기기(브라우저)에만 저장됩니다.
- **설치형 웹앱(PWA)** — GitHub Pages로 배포 후 휴대폰/PC에서 "홈 화면에 추가" 또는 "설치"로 앱처럼 사용할 수 있습니다.

## 마감/초기화 규칙 (자세히)

- 각 영업일(월~금, 공휴일은 반영되어 있지 않습니다)의 리포트는 **다음 영업일 오전 7시**까지 수정할 수 있습니다.
  - 예: 금요일 리포트는 토·일요일을 건너뛰어 **월요일 오전 7시**까지 계속 수정 가능합니다.
  - 오전 7시가 지나면 그 리포트는 자동으로 잠기고(조회 전용), 새 영업일의 빈 리포트가 열립니다.
- 화면 상단의 날짜 버튼(●작성중 / 마감)으로 지난 영업일 리포트를 조회할 수 있으며, 잠긴 리포트는 수정은 안 되지만 **PDF 저장·메일 발송은 그대로 가능**합니다(늦게 재전송이 필요한 경우 대비).
- 보관 기간은 작성 중인 리포트 포함 총 4영업일이며, 그보다 오래된 로컬 데이터는 자동으로 정리됩니다.
- 앱을 켜둔 상태로 오전 7시를 넘기면 1분 주기로 자동 체크되어 별도 새로고침 없이도 잠금/초기화가 반영됩니다.

## 메일 발송 방식 (EmailJS 없이, Gmail 앱 기반)

브라우저(웹앱)는 메일 서버에 직접 접속해 메일을 "자동 발송"할 수 없습니다. 이 앱은 대신 다음과 같이 동작합니다.

1. "메일 발송" 버튼 클릭
2. PDF가 자동으로 다운로드됨 (기기의 기본 다운로드 폴더 — Android/iOS 모두 Gmail의 "파일 첨부" 화면이 기본으로 보여주는 바로 그 위치입니다)
3. 받는 사람 / 제목 / 안내 문구가 채워진 Gmail 작성 화면이 자동으로 열림
4. "파일 첨부"에서 방금 다운로드된 PDF를 선택 → 보내기

**보내는 사람 주소**는 앱이 지정하는 게 아니라, 그 순간 휴대폰 Gmail 앱에 로그인되어 있는 계정이 사용됩니다. `kskim01@shinhan.com`으로 보내려면 휴대폰의 Gmail 앱 자체에서 그 계정으로 로그인해 두시면 됩니다. 설정 화면의 "보내는 사람 이메일"은 메일 본문에 참고용으로만 표시됩니다.

## 로그인 / 사용자 구분

앱을 열면 사번 ID 5자리를 입력해야 체크리스트에 들어갈 수 있습니다. 현재 등록된 사용자:

| ID | 이름 |
|---|---|
| 71213 | 김광석 |
| F1211 | 남윤호 |
| J0919 | 최한민 |
| Q0103 | 박인석 |
| Q0601 | 장동현 |

- 로그인하면 그 기기에 저장되어, 이후에는 자동으로 로그인된 상태로 열립니다. 상단의 "전환" 버튼을 누르면 로그아웃되고 다시 ID를 입력할 수 있습니다.
- 항목의 No. of Count / Checking Result를 바꾸면, 그 항목의 "최종수정자" 칸에 로그인한 사용자 이름이 자동으로 기록됩니다(화면 전용, PDF에는 포함되지 않음).
- **주의**: 이 로그인은 무단 접근을 가볍게 막는 용도입니다. ID 목록이 앱 코드(index.html) 안에 그대로 들어 있어서, 개발자도구로 소스를 보면 누구나 알아낼 수 있습니다. 민감한 정보를 지키는 진짜 보안이 필요하면 Firebase Authentication 같은 실제 인증 체계로 바꾸는 것을 권장하며, 필요하시면 도와드릴 수 있습니다.
- ID를 추가/변경하려면 `index.html` 안의 `USERS` 객체만 수정하면 됩니다.

## 팀원 간 같은 파일 공유를 위해 필요한 설정 — Firebase (선택)

**"김광석이 수정하면 남윤호·최한민·박인석·장동현이 바로 그 변경사항을 본다"는 실시간 공유는, 브라우저만으로는 불가능하고 아래 Firebase 설정이 되어 있어야 동작합니다.** 한 명(예: 관리자)이 아래 절차로 Firebase 프로젝트를 한 번만 만들고, 그 설정값을 5명 모두 각자의 앱 **설정** 탭에 동일하게 입력해 두면, 그 이후로는 별도 조치 없이 로그인해서 체크리스트를 열 때마다 자동으로 공유됩니다.

1. https://console.firebase.google.com 에서 무료 프로젝트 생성
2. 왼쪽 메뉴 **Firestore Database** > 데이터베이스 만들기 (테스트 모드로 시작 가능)
3. **프로젝트 설정(톱니바퀴) > 일반 > 내 앱 > 웹 앱 추가** 후 나오는 `firebaseConfig` 객체(JSON)를 복사
4. 앱의 **설정** 탭 > "팀 공유(Firebase)"에 그대로 붙여넣고 저장
5. **같은 firebaseConfig 값을 5명 모두 동일하게 입력**하면, 작성 중인 리포트를 실시간으로 함께 작성하게 됩니다. 지난 영업일 리포트도 팀원 화면에서 함께 조회됩니다(단, 실시간 동기화는 작성 중인 리포트에만 적용됩니다).

> Firestore 보안 규칙: 사내 전용으로만 쓸 경우 테스트 모드(공개 읽기/쓰기)로 충분하지만, 외부 노출이 걱정되면 Firestore 규칙에서 팀 도메인/이메일 인증을 추가하는 것을 권장합니다. 필요하면 도와드릴 수 있습니다.

## GitHub Pages로 배포하기 (설치 가능한 웹앱 만들기)

이 폴더 전체(`index.html`, `manifest.json`, `sw.js`, `icons/`)를 본인 GitHub 계정의 새 저장소에 올리면 됩니다.

```bash
# 1) 이 폴더에서 그대로 실행
git init
git add .
git commit -m "Kardian daily checklist PWA"

# 2) GitHub에서 새 저장소를 만든 뒤 (예: kardian-checklist), 원격 주소 연결
git branch -M main
git remote add origin https://github.com/<본인계정>/kardian-checklist.git
git push -u origin main
```

3. GitHub 저장소 **Settings > Pages**로 이동
4. **Source**를 `Deploy from a branch`로, **Branch**를 `main` / `/(root)`로 선택 후 저장
5. 잠시 후 `https://<본인계정>.github.io/kardian-checklist/` 주소로 접속 가능

### 휴대폰/PC에 앱처럼 설치하기

- **iPhone (Safari)**: 위 주소 접속 → 공유 버튼 → "홈 화면에 추가"
- **Android (Chrome)**: 위 주소 접속 → 메뉴(⋮) → "앱 설치" 또는 "홈 화면에 추가"
- **PC (Chrome/Edge)**: 주소창 오른쪽의 설치 아이콘 클릭

## 파일 구성

kardian-checklist/

## 참고 / 제한사항

- 체크리스트 데이터는 기본적으로 브라우저의 localStorage(기기별 저장)에 저장되며, Firebase를 연결하면 팀 전체가 공유하는 클라우드 저장소에 실시간 저장됩니다.
- PDF는 원본 엑셀 양식과 동일한 레이아웃으로 재구성되어 A4 한 페이지에 맞춰 생성됩니다.
- 영업일 계산은 토·일요일만 제외하며, 공휴일(설/추석 등)은 반영되어 있지 않습니다. 공휴일도 자동으로 건너뛰게 하려면 알려주시면 반영해 드릴 수 있습니다.
- 메일 첨부는 브라우저 보안 정책상 자동으로 붙지 않고, PDF 다운로드 후 Gmail에서 한 번 선택해 첨부하는 동작이 필요합니다.
